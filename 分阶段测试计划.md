# Daily Stock Analysis - 分阶段测试计划 (v4.0 架构升级)

本测试计划旨在分步骤、安全地验证“多专家共识架构”（Multi-Expert Ensemble Architecture）的集成情况。每完成一个阶段，必须进行代码提交和测试日志（test-log.md）的更新。

---

## 阶段一：Gemini CLI Headless 基础集成 (✅ 已完成)
- **目标**：验证在项目中通过子进程无头调用 `gemini-cli` 的可行性。
- **验收标准**：
  - [x] `-p` 参数能进入非交互模式。
  - [x] Stdin 管道数据能成功注入大段上下文。
  - [x] 成功解析非标准 JSON 输出（清理反引号）。

---

## 阶段二：单一“大师”节点验证 (✅ 已完成)
- **目标**：验证从 `AI-Hedge-Fund` 提取的大师 Prompt 能在当前项目中正常生成结构化信号。
- **验证结果**：`tests/test_single_expert.py` 证明巴菲特专家模型能准确输出 JSON 格式的价值投资逻辑。

---

## 阶段三：并发调用与共识引擎验证 (✅ 已完成)
- **目标**：验证并发启动多个 `gemini-cli` 实例的稳定性，以及共识引擎（Consensus Aggregator）的聚合逻辑。
- **验证结果**：`tests/test_ensemble_consensus.py` 成功拉起 6 个并发/串发进程。达里奥（Dalio）成功平衡了黄仁勋与塔勒布的意见冲突。

---

## 阶段四：战绩追踪与动态权重验证 (✅ 已完成)
- **目标**：验证 `analyst_track_record` 数据库存储及基于胜率的置信度调整公式。
- **验证结果**：`src/services/performance_tracker.py` 实现了胜率加权算法。通过 `scripts/backfill_performance.py` 成功模拟了专家胜率对最终决策权的影响。

---

## 阶段五：全量数据接入与 WebUI 渲染 (⏳ 待进行)
- **目标**：将新的分析引擎与现有的 A 股/港股 Data Provider 以及前端界面打通。
- **任务**：
  1. 确保新闻（News/Sentiment）和基本面（Fundamentals）数据被正确获取并拼接到上下文中。
  2. 更新 WebUI 界面，展示“雷达图”、“大师投票盘”和“胜率榜单”。
- **验收标准**：
  - 用户在 Web 界面输入 A 股/港股代码后，能清晰看到技术面、基本面、新闻面 3 位以上大师的独立意见。
  - 共识报告呈现合理。
